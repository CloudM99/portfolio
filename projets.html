Site commercial informatique

Objectif

Dans le cadre de ce projet, j'ai développé un site internet commercial dédié à la vente de matériel informatique. L'objectif principal était de créer une plateforme intuitive et fonctionnelle permettant aux utilisateurs d'acheter facilement divers produits informatiques. Le site inclut une interface de gestion des articles pour les administrateurs, leur permettant de créer, supprimer et modifier les articles disponibles à la vente. De plus, j'ai intégré une fonctionnalité permettant d'appliquer des remises sur certains articles, offrant ainsi une flexibilité dans la gestion des promotions.

Technologies utilisées

Pour réaliser ce projet, j'ai utilisé une variété de technologies et d'outils. Le développement a été effectué en utilisant le langage PHP, avec le framework Symfony pour structurer l'application. J'ai utilisé Doctrine ORM pour gérer les interactions avec la base de données MySQL, et Twig comme moteur de template pour générer les pages web dynamiquement. Le front-end a été optimisé grâce à Webpack Encore et Node.js, tandis que Symfony UX et Stimulus ont été utilisés pour améliorer l'expérience utilisateur. Hotwire et Turbo ont également été intégrés pour rendre l'application plus réactive sans nécessiter beaucoup de JavaScript.

Environnement de Développement

L'environnement de développement a été configuré sur un ordinateur Windows 11, en utilisant XAMPP pour simuler un serveur local. PhpStorm a été mon environnement de développement intégré (IDE) principal, offrant des fonctionnalités avancées pour l'édition de code et le débogage. PhpMyAdmin a été utilisé pour gérer la base de données MySQL de manière visuelle et efficace.

Gestion de Version et Hébergement

Pour assurer une gestion efficace du code source et garantir la sécurité de mes données, j'ai utilisé GitHub comme plateforme de versionnage. GitHub m'a permis de suivre les modifications apportées au code, de maintenir un historique complet des versions du projet et de faciliter la gestion des différentes branches de développement. En complément, j'ai choisi OVH pour héberger le site internet et la base de données. OVH offre des solutions d'hébergement fiables et performantes, assurant ainsi une disponibilité optimale du site pour les utilisateurs. Cette combinaison de GitHub pour le versionnage et OVH pour l'hébergement a été essentielle pour garantir la stabilité et la pérennité du projet.

Ressources Documentaires

Afin de mener à bien ce projet, je me suis appuyé sur diverses ressources documentaires. La documentation officielle de PHP et Symfony a été essentielle pour comprendre les meilleures pratiques et les fonctionnalités avancées. Les cours disponibles sur Moodle m'ont également fourni des connaissances théoriques et pratiques précieuses. La documentation de XAMPP a été utile pour configurer et optimiser l'environnement de développement local.

Mon Projet Professionnel

Suseta

Dans le cadre de mon alternance, j'ai travaillé sur des applications de gestion des mesures conservatoires. Mon manager m'a confié la mission de créer un nouvel outil reprenant les fonctionnalités d'un outil existant, dont le serveur va être décommissionné. Ce site était utilisé par une équipe d'une dizaine de personnes à travers la France pour formater les données nécessaires à la gestion des mesures conservatoires.

Les mesures conservatoires sont appliquées lorsque le client n'a pas payé ses factures pour des services donnés, entraînant la suspension de ces services. Une fois le règlement effectué, les services sont rétablis. C'est en m'inspirant de ces mécanismes que j'ai nommé l'outil "Suseta", qui reprend "Sus" pour "Suspension" et "eta" pour "Rétablissement".

Pour des raisons de confidentialité, je ne peux pas partager le code source de l'application. Cependant, je peux vous partager un aperçu de Suseta. Il y a 4 pages accessibles aux utilisateurs, et une page d'administration.

La première page "Traitement par service" permet à l'utilisateur de choisir l'action à effectuer, le service concerné et de renseigner la référence nécessaire.

La deuxième page "Consultation des données" permet de consulter les références qui ont étés renseignés et donc stockés dans une table temporaire. L'utilisateur peut également rentrer une référence présente dans cette liste pour annuler le traitement.

La troisième page "Génération des fichiers" permet de générer les fichiers nécessaires au traitement des mesures. Le traitement diffère selon le service et l'action choisie. Les fichiers générés devront ensuite être déposés sur les applications spécifiques aux services. Lorsque l'utilisateur clique sur le bouton "Générer", il est redirigé vers une page de confirmation. La table temporaire est vidée et les mesures sont historisées dans une autre table.

La quatrième page "Historique" permet de consulter les mesures qui ont été traitées. Les mesures sont affichés par fichier et par date de traitement.
